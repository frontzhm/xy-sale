# 微信 JSSDK 接入条件与方案（聊天图片选择）

本文用于评估并落地「在微信内打开页面时，上传图片可稳定从聊天会话选择」能力。  
结论先行：若要稳定支持聊天来源，建议使用微信 JSSDK（`wx.chooseMessageFile`），不要只依赖原生 `input[type=file]`。

## 1. 你要的能力是什么

目标能力：

- 页面在微信内置浏览器打开
- 点击上传按钮后，可从微信聊天会话里选图片
- 选中后上传到你现有服务端（再写入 OSS / 存储）

非目标（原生 file input 无法保证）：

- 强制所有机型都出现「聊天图片」入口
- 完全控制微信系统选择器 UI

## 2. 接入前置条件（必须满足）

### 2.1 账号与域名

- 需要一个可用的微信公众号（服务号/订阅号，能使用 JS-SDK）
- 页面域名已备案且可公网 HTTPS 访问
- 在公众号后台配置 **JS 接口安全域名**（必须与页面实际访问域名一致）

### 2.2 页面运行环境

- 必须在微信内置浏览器打开（非外部 Safari/Chrome）
- URL 不能是 `localhost`、内网 IP 或未备案测试域名
- 最终页面 URL 与签名时使用的 URL（去掉 hash）一致

### 2.3 服务端能力

- 服务端可安全保存 `appId`、`appSecret`
- 服务端可获取并缓存：
  - `access_token`
  - `jsapi_ticket`
- 服务端可提供「签名接口」返回 `nonceStr/timestamp/signature`

## 3. 微信后台配置清单

在公众号后台完成以下配置：

- 设置与开发 -> 基本配置：拿到 `AppID`、`AppSecret`
- 设置与开发 -> 功能设置 -> JS 接口安全域名：填写你的前端域名
- 确保生产环境 URL 与安全域名严格匹配（含子域名）

建议：

- 生产、预发使用不同公众号或至少不同域名策略，避免签名混用

## 4. 技术实现总览（Next.js）

推荐分 3 层：

- 服务端：提供 `GET /api/wechat/jssdk-sign?url=...`
- 前端：页面加载后执行 `wx.config(...)` + `wx.ready(...)`
- 上传层：调用 `wx.chooseMessageFile`，拿到文件后走你现有上传接口

## 5. 签名接口需要做什么

签名核心规则：

- 签名对象是「当前页面完整 URL（不含 # 后部分）」
- 字段：`jsapi_ticket`、`noncestr`、`timestamp`、`url`
- 算法：SHA1

服务端签名接口返回示例：

```json
{
  "appId": "wx123...",
  "timestamp": 1714200000,
  "nonceStr": "random-string",
  "signature": "sha1-result",
  "expiresIn": 7000
}
```

缓存建议：

- `access_token`、`jsapi_ticket` 都按过期时间缓存（例如预留 5 分钟提前刷新）
- 不要每次页面请求都实时向微信拉 token/ticket

## 6. 前端接入流程

### 6.1 加载 JSSDK

可使用官方脚本（例如 `https://res.wx.qq.com/open/js/jweixin-1.6.0.js`），或 npm 包方案。  
重点是确保 `wx` 可用，再执行配置。

### 6.2 初始化

前端流程：

1. 取当前页面 URL（`window.location.href.split('#')[0]`）
2. 请求你自己的签名接口
3. 调用 `wx.config`
4. 在 `wx.ready` 后才允许调用 `chooseMessageFile`

`jsApiList` 至少包含：

- `chooseMessageFile`

### 6.3 选择聊天图片并上传

典型流程：

1. `wx.chooseMessageFile({ count, type: 'image' })`
2. 获取返回文件对象（不同版本字段略有差异）
3. 把文件交给你已有上传 API（如 `FormData`）
4. 成功后回填业务表单

## 7. 与当前项目的适配点（xy-sale）

你当前项目在新建衣服档案里使用的是 Antd `Upload` + 原生 file input。  
若接入 JSSDK，建议策略：

- 微信内：显示「从微信聊天选择」按钮（JSSDK）
- 非微信：继续使用现有 `Upload`（浏览器原生）

这样可以：

- 保持桌面端与普通手机浏览器不受影响
- 在微信内获得稳定聊天来源入口

## 8. 常见坑（高频）

- 签名 URL 不一致（最常见）
  - 页面是 `https://a.com/x?y=1`
  - 你签名用了 `https://a.com/x`（少 query）或用了 hash
- 域名不在 JS 安全域名白名单
- 页面发生重定向，导致签名时 URL 与实际 URL 不同
- token/ticket 缓存过期控制错误（时钟漂移、没提前刷新）
- iOS 与 Android 微信上字段表现差异未兼容

## 9. 安全与合规建议

- `appSecret` 仅放服务端，前端绝不暴露
- 签名接口只对受信来源开放（至少做基础防刷）
- 上传接口继续做服务端校验（文件类型、大小、恶意内容）
- 如涉及隐私图片，补充存储生命周期与访问权限策略
